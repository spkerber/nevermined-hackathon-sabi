Sabi

Verification-as-a-Service. Submit geolocated questions, get photo-verified answers from on-the-ground humans wearing Ray-Ban Meta smart glasses.

Nevermined x402 Payments Real-time WebSocket Cloudflare Workers TypeScript SDK
Quick Start SDK CLI API Reference Concepts

Get Started in 60 Seconds

CLI (npx)

# Set your Nevermined API key
export NVM_API_KEY="nvm_..."
export SABI_API_URL="https://sabi-backend.ben-imadali.workers.dev"

# Create a verification
npx sabi verify \
  "Is the coffee shop on 5th Ave open?" \
  --lat 37.7749 --lng -122.4194

# Watch it happen
npx sabi watch <job-id>

TypeScript SDK

// npm install sabi
import { SabiClient } from "sabi";

const sabi = new SabiClient({
  apiUrl: "https://sabi-backend.ben-imadali.workers.dev",
  nvmApiKey: process.env.NVM_API_KEY,
});

const { job } = await sabi.createVerification({
  question: "Is the coffee shop open?",
  targetLat: 37.7749,
  targetLng: -122.4194,
});

console.log("Job:", job.id);

Prerequisites

  1. Get a Nevermined API key from sandbox.nevermined.app
  2. Order the Sabi plan in the Nevermined App to get credits
  3. Set NVM_API_KEY and SABI_API_URL as environment variables (or use sabi config)

How It Works

  1. You submit a question + GPS coordinates. Payment (1 Nevermined credit) is handled automatically.
  2. A nearby human verifier with Ray-Ban Meta glasses accepts the job.
  3. The verifier goes on-site, captures timestamped photos (1 every 5 seconds), and answers vocally.
  4. You receive the artifact: original question + transcribed answer + photo evidence.

watchJob(jobId)

const watcher = client.watchJob(result.job.id);

watcher.on("state", (state) => {
  console.log(`Status: ${state.status}, Frames: ${state.frames.length}`);
  if (state.status === "verified") {
    console.log("Answer:", state.job.answer);
    watcher.close();
  }
});

watcher.on("error", (err) => console.error(err));
watcher.on("close", () => console.log("Done"));

const artifact = await client.getArtifact(jobId);

console.log("Question:", artifact.question);
console.log("Answer:", artifact.answer);
console.log("Transcript:", artifact.transcript);
for (const frame of artifact.frames) {
  console.log(`  Photo: ${frame.url} at ${new Date(frame.timestamp)}`);
}

Error Handling

import { SabiClient, PaymentRequiredError, PaymentFailedError, ApiError } from "sabi";

try {
  await client.createVerification({ ... });
} catch (err) {
  if (err instanceof PaymentRequiredError) {
    console.log("Set NVM_API_KEY or order the plan");
  } else if (err instanceof PaymentFailedError) {
    console.log("Token acquisition failed:", err.message);
  } else if (err instanceof ApiError) {
    console.log(`API error ${err.status}:`, err.message);
  }
}

Payment Flow (Nevermined x402)

Every verification job costs 1 Nevermined credit. The SDK handles this automatically, but here's the underlying flow:

  1. GET /api/config returns the nvmPlanId, nvmAgentId, and nvmEnvironment.
  2. Your NVM_API_KEY is used to call POST {nvmBackend}/api/v1/x402/permissions with the plan details. This returns an accessToken.
  3. POST /api/verifications is called with payment-signature: <accessToken>. The backend verifies the token, settles (burns) 1 credit, and creates the job.
No Nevermined SDK needed. The payment flow uses direct REST calls. The sabi package has zero Nevermined dependencies.

Job Lifecycle

  connecting ──accept──> accepted ──start──> in_progress ──end──> verified
       ^                    |                     |
       └────cancel──────────┘                     |
                                                  └──cancel──> connecting

  (any state) ──archive──> (removed from registry)
StatusMeaning
connectingWaiting for a verifier to accept
acceptedVerifier claimed the job
in_progressVerifier is on-site capturing evidence
verifiedComplete. Artifact (answer + photos) available.
cancelledCancelled by verifier or requester

Architecture

Sabi runs on Cloudflare Workers with the Agents SDK:

ComponentTechnology
API + routingCloudflare Worker (fetch handler)
Job stateDurable Object (VerificationAgent) with SQLite
Job registryDurable Object (JobRegistry) for listing
Photo storageCloudflare R2
Real-time updatesWebSocket via Agents SDK
PaymentNevermined x402 (REST API)

Multi-Agent Architecture

AgentRole
SellerSabi backend. Validates payment, matches verifiers, delivers artifacts.
